There are 3 kinds of lambda terms:
* Variables, written '', where  is a variable name.

* Abstraction, written '位. ' where  is a variable name and  is a lambda
  term. The abstraction binds  in . Abstraction reaches as far right as
  possible; '位x. x y' is the same as '位x. (x y)', not '(位x. x) y'.

* Application, written ' ' where  and  are both lambda terms. Application
  is left associative; 'a b c' is the same as '(a b) c', not 'a (b c)'.

Variables are considered either free or bound. Given a lambda term , a
variable  is free in  if it is not bound, and  is bound in  if all
occurrences of  appear within abstractions that bind .

For example, the variable x is bound in the lambda term '位x. x' as x only
appears within the abstraction '位x. x' which binds it. The variable x is also
bound in '位y. y' by vacuous truth. Conversely, the variable x is free in both
'x' and '位y. x' since x has occurrences outside of any abstractions that bind x.

Lastly, a lambda term  is considered closed if no variables are free in .
This interpreter requires that its input lambda terms are closed and will
produce an error if you give it input that is not closed.
